# דפדף

בונה דפי נחיתה בעברית בעזרת AI. המשתמש מחדד הצעה, קהל והוכחות בבריף של שלושה שלבים, בוחר מטרה וכיוון חזותי, ומקבל דף שניתן לערוך בסביבת עבודה ולהוריד כקובץ HTML עצמאי.

## חוויית המוצר

- **בריף אסטרטגי** — שם העסק והתחום, הצעה מרכזית, קהל יעד, פירוט הדרך שבה העסק עוזר, הוכחות אמיתיות ומסר קמפיין קיים.
- **בחירת כיוון** — מטרה עסקית, אופי חזותי וחומרי מותג אופציונליים. מנוע העיצוב מתאים ארכיטיפ, קומפוזיציית Hero וסדר מקטעים לתחום ולמידע שהוזן.
- **סביבת עבודה** — ניווט לפי מקטעי הדף, עריכת טקסט על הקנבס, שכתוב ממוקד, שלושה כיווני עיצוב, תצוגת מובייל, הוספת מקטעים, בדיקת המרות ושמירה מקומית.
- **אמת בפרסום** — המודל והעורך אינם אמורים להמציא לקוחות, עדויות, מספרים, הסמכות או תוצאות. מקטעי הוכחה מבוססים רק על מידע שהמשתמש מסר, ושדות מספריים שנוספים ידנית מסומנים כדרושים לעריכה ואימות.

## דרישות

- Node.js 20 ומעלה
- פרויקט Vercel
- מפתח OpenAI API
- בסיס Upstash Redis להגבלת שימוש משותפת בין פונקציות Serverless

## משתני סביבה

העתיקו את `.env.example` והגדירו:

- `OPENAI_API_KEY` — מפתח OpenAI בצד השרת בלבד.
- `OPENAI_MODEL` — אופציונלי. ברירת המחדל היא `gpt-5.6-luna`.
- `UPSTASH_REDIS_REST_KV_REST_API_URL` — כתובת REST שנוצרת אוטומטית בחיבור Upstash for Redis דרך Vercel.
- `UPSTASH_REDIS_REST_KV_REST_API_TOKEN` — token לכתיבה שנוצר אוטומטית בחיבור Upstash for Redis דרך Vercel.
- `RATE_LIMIT_SALT` — ערך אקראי המשמש לגיבוב כתובות IP במפתחות ההגבלה.
- `UNSPLASH_ACCESS_KEY` — אופציונלי. מפתח Access Key של אפליקציית Unsplash, בצד השרת בלבד. אם לא מוגדר, גלריית התמונות האוטומטית פשוט לא מוצגת כשלא הועלו תמונות.

אין להשתמש ב־`UPSTASH_REDIS_REST_KV_REST_API_READ_ONLY_TOKEN`, מפני שמנגנון הגבלת השימוש צריך לכתוב מונים למסד.

אם משתני Upstash אינם מוגדרים, נקודות הקצה של ה-AI מחזירות 503 במקום לפעול ללא הגנה.

## ספק AI

המערכת משתמשת ב־OpenAI Responses API עם Structured Outputs מסוג JSON Schema. הפלט עדיין עובר validation בצד השרת לפני שהוא נשלח לדפדפן. אפשר להחליף מודל דרך `OPENAI_MODEL` בלי שינוי קוד.

## הרצה ובדיקות

```bash
npm test
npm run lint
npm run format:check
npm run check
```

את האתר הסטטי ניתן לפתוח דרך שרת מקומי. פונקציות `/api` מיועדות לסביבת Vercel או סביבת פיתוח שמדמה אותה.

## הגנות שימוש

- יצירת דף: עד 5 בקשות לשעה לכל IP.
- שכתוב מקטע: עד 20 בקשות לשעה לכל IP.
- לכל endpoint יש scope נפרד.
- בקשות חורגות מקבלות HTTP 429 ו-`Retry-After`.
- קלט ותשובות AI עוברים validation והגבלת גודל.
- לכל קריאת AI יש timeout של 30 שניות ו-retry יחיד רק כאשר הפלט אינו תקין.

## דפדף פלוס — כלים וערכת קמפיין

שכבות `js/plus.js`, `js/plus-corrections.js` ו־`css/plus.css` מוסיפות כלים אופציונליים שמופעלים מתוך סרגל התוצאה ונכנסים גם לתצוגה המקדימה וגם לקובץ המיוצא:

- **הודעת שעות שבת** — מחשבת שקיעה לפי תאריך ועיר בישראל, משתמשת בזמן הדלקת נרות מקובל לעיר ובמספר דקות הניתן להגדרה לצאת שבת, ומעדכנת את הבאנר גם כאשר הדף נשאר פתוח. החישוב אסטרונומי ויש לאמת זמנים ומנהגים לפני פרסום.
- **כלי עזר לנגישות** — הגדלת טקסט, ניגודיות, הדגשת קישורים ועצירת אנימציות עם שמירת העדפה ב־localStorage. הכלים אינם אישור לעמידה בתקן ואינם תחליף לבדיקת נגישות מקצועית או ייעוץ משפטי.
- **כפתור WhatsApp צף** — משתמש במספר שהוזן בחלונית "לידים ומדידה" עם הודעת פתיחה מוכנה.
- **סרגל פעולות למובייל** — חיוג, WhatsApp וניווט ב־Waze כאשר הוגדר יעד אמיתי.
- **מסרים מתחלפים** — בועות עם טקסטים שבעל העסק מזין. אין להציג רכישות, ביקורות או פעילות שלא התרחשו בפועל.
- **טיימר יומי** — ספירה לאחור עד חצות לפי `Asia/Jerusalem`. יש להפעיל אותו רק כאשר קיימת הטבה אמיתית שמסתיימת בזמן המוצג.

בנוסף, **ערכת קמפיין** בונה מהתוכן הנוכחי של הדף, כולל עריכות ידניות, נכסי פרסום מוכנים לבדיקה ולהעתקה: מודעת פייסבוק/אינסטגרם, כותרות ותיאורים לגוגל בהגבלות התווים, הודעת WhatsApp, SMS וביו לאינסטגרם. הכול בצד הלקוח, ללא קריאת AI נוספת.

הייצוא מוסיף meta description ותגיות Open Graph שנבנות מתוכן ה־Hero, ו־favicon מהלוגו שהועלה. הגדרות הבוסטרים נשמרות ב־localStorage.

## ייצוא, לידים ומדידה

לאחר יצירת הדף ניתן לפתוח את חלונית "לידים ומדידה" ולהזין WhatsApp, אימייל, URL חיצוני, כתובת מאובטחת לקליטת טפסים ומזהה Google Tag Manager. הקובץ המיוצא כולל רק פעולות שהוגדר להן יעד אמיתי:

- WhatsApp או URL הופכים לקישור פעיל.
- כתובת טופס מסוג HTTPS הופכת את אזור ההנעה לפעולה לטופס POST עם שם, טלפון ואימייל אופציונלי.
- אימייל משתמש זמנית בטופס `mailto:` ופותח את תוכנת הדואר של המבקר.
- מזהה `GTM-…` תקין מטמיע את תגי Google Tag Manager בתוך הקובץ המיוצא.
- ללא יעד פעיל, כפתורי CTA וטופס שאינם עובדים מוסרים מהייצוא.

`mailto:` אינו backend לטפסים. כתובת קליטת הטופס צריכה להגיע משירות מאובטח שתומך ב־POST רגיל, ולכלול טיפול מתאים בהסכמה לפרטיות ובאחסון מידע.

## מצב מסחרי ומגבלות נוכחיות

המערכת נמצאת בבטא פתוחה. אין עדיין חשבונות משתמשים, סליקה, שמירת דפים בענן, דומיינים אישיים, ניהול לידים פנימי או מערכת אנליטיקה בתוך דפדף. אין להציג מסלולים בתשלום או הבטחות ליכולות אלה כאילו הם זמינים לרכישה.

אין לטעון שהכלים הם "ראשונים בעולם", שאין אותם אצל מתחרים, או שכלי העזר לנגישות מבטיחים עמידה בחוק. טענות שיווקיות ומשפטיות דורשות ביסוס ובדיקה נפרדת.
